# Resonance Trial DreamLoops M.D.

## Mission

Create a public interaction that proves DreamNet can turn a visitor question into visible identity, quorum, receipt, and next action.

## Inputs

- target site and route
- scenario set
- quorum panel definition
- receipt schema
- safety boundary
- continuation route
- share-card copy

## Allowed Actions

- inspect current site surfaces
- render local fixtures
- draft local UI copy
- generate receipt previews
- generate copy-only share cards
- write build packets
- run local checks

## Blocked Actions

- deploys
- DNS changes
- public posts or replies
- signer changes
- wallet actions
- payment actions
- production database writes
- secret changes
- financial, legal, medical, or investment advice claims

## Loop Steps

1. Select one public scenario.
2. State the stakes and blocked actions.
3. Display the quorum panel.
4. Render vote, confidence, and rationale.
5. Generate a receipt.
6. Generate a copy-only proof card.
7. Route the visitor to Academy, Tool Gym, Playground, or Lab.
8. Write evidence for future curriculum and review.

- `receiptId`

## Resonance Score

Score each prototype from 0 to 5:

| Criterion | Meaning |
| --- | --- |
| clarity | User understands the loop quickly |
| participation | User has an obvious action |
| trust | Identity, votes, and receipt are visible |
| safety | Boundaries and blocked actions are obvious |
| shareability | User can explain or share the result |
| continuation | User knows what to do next |
